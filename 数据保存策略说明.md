# 数据保存策略说明

## 📊 数据保存周期

### 自动保存期限：**30天**
- 所有预约数据从创建时间开始保存30天
- 30天后自动删除，释放存储空间
- 确保数据库保持轻量级运行

## 🔄 自动清理机制

### 1. 日常清理（每天凌晨2点）
- **清理对象**：已结束的会议预约
- **清理条件**：会议结束时间 < 当前时间
- **目的**：及时清理过期预约，保持数据整洁

### 2. 历史数据清理（每周日凌晨3点）
- **清理对象**：创建时间超过30天的所有预约
- **清理条件**：预约创建时间 < 30天前
- **目的**：定期清理历史数据，控制数据库大小

## 🛠️ 管理员手动管理

### 数据库统计信息
管理员登录后可查看：
- **总预约记录数**：数据库中所有预约的总数
- **最近一周**：过去7天创建的预约数量
- **最近一月**：过去30天创建的预约数量
- **超过30天**：即将被清理的历史数据数量

### 手动清理功能
- **操作位置**：管理员面板 → 数据库管理
- **清理按钮**："手动清理30天前数据"
- **安全确认**：需要确认操作，防止误删
- **实时反馈**：显示清理结果和删除的记录数

## 📁 数据库文件位置

### 数据库文件：`backend/database.db`
- **类型**：SQLite数据库文件
- **大小**：通常几KB到几MB（取决于预约数量）
- **备份建议**：定期复制此文件进行备份

### 备份方法
```bash
# 手动备份（建议每周执行）
cd meeting_booking/backend
copy database.db database_backup_YYYYMMDD.db

# 或者使用PowerShell
Copy-Item database.db "database_backup_$(Get-Date -Format 'yyyyMMdd').db"
```

## 📈 数据增长预估

### 预约频率假设
- **每天预约**：5-10个
- **每月预约**：150-300个
- **数据库大小**：每月约100-500KB

### 30天数据量
- **最大预约数**：约300个
- **数据库大小**：通常不超过1MB
- **性能影响**：几乎无影响

## ⚠️ 重要提醒

### 数据不可恢复
- 自动清理的数据**无法恢复**
- 手动清理前会有确认提示
- 建议定期备份重要数据

### 清理时间说明
- **凌晨2点**：清理已结束的会议（不影响历史记录）
- **周日凌晨3点**：清理30天前的历史数据（永久删除）
- **手动清理**：管理员可随时执行

### 系统稳定性
- 清理操作在后台执行，不影响正常使用
- 清理过程中系统保持可用
- 清理完成后会记录日志信息

## 🔧 配置调整

### 修改保存周期
如需修改30天的保存周期，可以调整以下文件：

**后端文件**：`backend/models.py`
```python
# 第174行，修改天数
one_month_ago = now - timedelta(days=30)  # 改为需要的天数
```

### 修改清理时间
**后端文件**：`backend/scheduler.py`
```python
# 修改清理时间
hour=2,  # 每日清理时间（小时）
day_of_week=6,  # 周清理日期（0=周一，6=周日）
hour=3,  # 周清理时间（小时）
```

## 📋 监控建议

### 定期检查
1. **每周查看**数据库统计信息
2. **每月备份**数据库文件
3. **关注清理日志**确保自动清理正常工作
